# 心契同眠语音与台本整理施工方案

## 目标

本次重构的目标不是让每一句台词都“很有戏”，而是让语音更像真人在自然说话：

- 日常交互以普通、温柔、开心为主。
- 表白、结婚、亲吻等明确高情绪剧情才使用害羞。
- 悲伤、生气、惊讶、疲惫只在文本语义真的需要时出现。
- 台词和旁白在 UI 上分开显示，不再把旁白紧贴在女仆台词后面。
- 语音生成、试听、重生成都能按剧情分组管理。

## 台本显示规则

每条剧情文本可以继续用一个 `text` 字段写成多行：

```text
女仆说的话
旁白描写
```

运行时显示时会按以下规则拆开：

- 第一行作为女仆台词，显示女仆名字。
- 后续行作为旁白，显示为独立旁白文本。
- 语音合成只使用第一行女仆台词，不合成旁白。
- 如果 `speaker` 明确为空且文本只有旁白，则不显示女仆名字。

## 情绪标注规则

语音情绪的默认分布：

- `normal`：默认情绪，日常对话优先使用。
- `gentle`：安慰、陪伴、膝枕、婚后平稳亲密。
- `happy`：轻快、开心、收到礼物、孩子成长。
- `shy`：只给表白、结婚、初次亲吻、明显脸红的少量句子。
- `serious`：誓约、承诺、重要说明。
- `sad`：低落、委屈、心情差。
- `tired`：学习疲惫、夜间困倦。
- `question`：疑惑、确认、犹豫。
- `teasing`：调侃、开玩笑。
- `angry`：冒犯、拒绝、明显不满。
- `surprised`：突发消息、意外反应。

禁止再用“节点名里有 marriage / 誓 / shy 就全部归害羞”的粗暴规则。

## 语音分组

扫描器会给每条语音写入分组，便于 UI 管理：

- `入口与日常`
- `沟通聊天`
- `表白`
- `结婚`
- `亲吻`
- `拥抱`
- `摸头`
- `膝枕`
- `礼物`
- `怀孕与育儿`
- `小女仆互动`
- `工作与学习`
- `系统反馈`

## UI 功能

主交互界面右下角增加麦克风按钮：

- 当前句子有预生成语音时显示绿色。
- 当前句子没有语音时显示灰色。
- 点击后重新播放当前句语音。

模组设置内增加语音管理入口：

- 查看当前台本的语音分组。
- 查看每组总数、已生成、失败、缺失。
- 选中单句后试听。
- 对单句或整个分组标记为重新生成。
- 合成队列支持暂停、中断、保存进度。

## 落地顺序

1. 修 UI 文本拆分和说话人显示。
2. 修 TTS 扫描器，只合成女仆台词并写入分组。
3. 收窄情绪映射，减少害羞使用。
4. 增加麦克风重播按钮。
5. 增加语音管理数据结构和设置页入口。
6. 构建主模组和 TTS 附属。
